## 1. 文档说明

### 1.1 编写目的

本文档面向产品、研发、测试、运营及业务方，统一描述"企业基础运营管理系统"中**用户管理**与**库存管理**两个核心模块的业务目标、功能范围、流程规则与验收标准，作为后续概要设计、详细设计、开发与测试的基线依据。

### 1.2 适用范围

- 一期建设：用户管理、库存管理两个核心模块
- 适用对象：产品经理、UI/UX 设计师、研发工程师、测试工程师、业务运营、仓库管理员、IT 运维

### 1.3 名词与缩略语

| 术语 | 全称/含义 |
|------|-----------|
| BRD | Business Requirements Document，业务需求文档 |
| RBAC | Role-Based Access Control，基于角色的访问控制 |
| SKU | Stock Keeping Unit，最小库存单位 |
| SPU | Standard Product Unit，标准化产品单元 |
| 入库单 | 记录商品入仓的单据（含采购入库、退货入库、调拨入库） |
| 出库单 | 记录商品出仓的单据（含销售出库、领用出库、调拨出库） |
| 安全库存 | 为应对需求/供应不确定性而设置的最低库存阈值 |

| 调拨 | 商品在不同仓库或库位之间的转移 |
| 盘点 | 实物清点并与系统账面对比、调整差异的过程 |

## 2. 项目概述

### 2.1 项目背景

随着公司业务规模扩张，原有 Excel 与多套独立工具并行的方式已暴露以下痛点：

1. **账号分散**：各系统账号独立，无统一身份与权限，员工入离职处理繁琐，存在权限残留风险。
2. **库存不清**：库存账实不一致，缺料/积压并存，缺乏实时数据支撑采购与销售决策。
3. **流程无审计**：关键操作（出入库、权限变更）缺乏完整日志，事后无法追溯。
4. **手工成本高**：盘点、报表、对账等高频操作依赖人工，效率低、错误率高。

### 2.2 业务目标

| 编号 | 业务目标 | 衡量指标（KPI） |
|------|----------|------------------|
| G1 | 建立统一账号与权限体系 | 全员账号统一率 100%；离职账号关闭 ≤ 1 个工作日 |
| G2 | 实现库存数据实时、准确 | 账实一致率 ≥ 99%；库存数据延迟 ≤ 1 分钟 |
| G3 | 关键操作可审计 | 用户、库存关键操作日志 100% 留痕，保留期 ≥ 1 年 |
| G4 | 提升仓库作业效率 | 入库/出库单平均处理时长降低 ≥ 40% |
| G5 | 降低缺货与积压 | 缺货率 ≤ 2%；滞销库存占比下降 ≥ 30% |

### 2.3 项目范围

**范围内（一期）：**

- 用户管理：注册、登录、信息维护、角色与权限、状态管理、操作审计
- 库存管理：商品档案、仓库/库位、入库、出库、调拨、盘点、预警、查询与报表

**范围外（一期不做，后续规划）：**

- 采购管理、销售订单、财务结算、客户关系管理（CRM）
- 供应链协同（与上游供应商系统对接）
- 移动端 App（一期仅 Web，后续规划 PDA/H5）
- 智能补货、需求预测等 AI 能力

### 2.4 干系人

| 角色 | 关注点 |
|------|--------|
| 业务负责人 | 业务目标达成、KPI 改善 |
| 仓库主管 | 库存准确性、作业效率、异常预警 |
| 仓库管理员 | 操作便捷性、单据流转流畅度 |
| 采购/销售人员 | 库存可见性、可用库存查询 |
| 系统管理员 | 用户/权限可控、安全合规 |
| 内审/合规 | 操作审计、权限分离、最小授权 |
| IT 研发/运维 | 系统可维护、可扩展、稳定 |

## 3. 总体设计

### 3.1 系统模块图（文字描述）

```
企业基础运营管理系统
├── 用户管理（UM）
│   ├── 账号管理
│   ├── 角色管理
│   ├── 权限管理（RBAC）
│   ├── 组织架构（部门）
│   └── 操作审计日志
└── 库存管理（IM）
    ├── 基础档案（商品、仓库、库位、单位、类别）
    ├── 库存业务（入库、出库、调拨、盘点）
    ├── 库存预警
    └── 库存查询与报表
```

### 3.2 系统角色定义

| 角色编码 | 角色名称 | 描述 | 默认权限范围 |
|----------|----------|------|--------------|
| ROLE_SUPER_ADMIN | 超级管理员 | 系统最高权限，仅 1～2 人 | 所有功能（含权限分配） |
| ROLE_SYS_ADMIN | 系统管理员 | 负责账号、角色、权限维护 | 用户管理全部、库存只读 |
| ROLE_WAREHOUSE_MGR | 仓库主管 | 审核出入库、调拨、盘点 | 库存管理全部、用户只读 |
| ROLE_WAREHOUSE_OP | 仓库管理员 | 录入出入库、盘点 | 库存业务录入、查询 |
| ROLE_PURCHASER | 采购员 | 提交入库申请、查看库存 | 入库录入、库存查询 |
| ROLE_SALES | 销售员 | 提交出库申请、查询可用库存 | 出库申请、库存查询 |
| ROLE_AUDITOR | 审计员 | 查看所有日志与报表 | 全模块只读 + 审计日志 |
| ROLE_USER | 普通用户 | 仅查看个人信息 | 个人中心 |

### 3.3 跨模块关系

- 库存管理的所有操作必须基于已登录的、具备相应权限的用户身份。
- 库存关键操作（入库、出库、盘点、调整）需写入操作日志，可追溯至具体用户。
- 角色变更后实时生效，正在登录的用户在下一次请求时按新权限校验。

## 4. 用户管理模块需求

### 4.1 模块目标

构建统一的用户身份与权限管理中心，覆盖"账号—角色—权限"三层模型，支持灵活的授权、严格的安全控制以及完整的操作审计，满足合规与运维需求。

### 4.2 功能清单

| 编号 | 功能名称 | 优先级 |
|------|----------|--------|
| F-UM-01 | 用户注册 | P0 |
| F-UM-02 | 用户登录与登出 | P0 |
| F-UM-03 | 用户信息维护 | P0 |
| F-UM-04 | 密码管理（修改、找回、强度策略） | P0 |
| F-UM-05 | 角色管理 | P0 |
| F-UM-06 | 权限管理 | P0 |
| F-UM-07 | 用户状态管理（启用/禁用/锁定） | P0 |
| F-UM-08 | 部门/组织架构管理 | P1 |
| F-UM-09 | 用户查询与批量操作 | P1 |
| F-UM-10 | 操作审计日志 | P0 |
| F-UM-11 | 登录安全策略（限频、验证码、双因素） | P1 |

### 4.3 详细功能需求

#### F-UM-01 用户注册

- **业务说明**：员工入职后由系统管理员创建账号，或开放有限的自助注册（可配置）。
- **功能要点**：
  1. 由管理员代建账号：录入工号、姓名、手机号、邮箱、所属部门、初始角色，系统生成初始密码并通过邮箱/短信下发。
  2. 自助注册（可选关闭）：填写工号、手机号、邮箱、姓名，提交后由管理员审核激活。
  3. 必填字段校验：工号唯一、手机号格式、邮箱格式、姓名非空。
  4. 重复性校验：工号、手机号、邮箱不可重复。
- **业务规则**：
  - 初始密码默认 12 位随机串，首次登录强制修改。
  - 创建成功后用户状态为"待激活"，激活后变为"启用"。

#### F-UM-02 用户登录与登出

- **登录方式**：账号密码登录（必选）；可配置开启短信验证码、企业微信/钉钉扫码、SSO（一期可不实现，预留接口）。
- **登录流程**：
  1. 输入账号（工号/手机号/邮箱任一）+ 密码。
  2. 校验账号是否存在、是否启用、是否锁定。
  3. 密码校验通过后下发会话 Token，记录登录日志。
  4. 失败时按"登录安全策略"处理（见 F-UM-11）。
- **登出**：用户主动登出或会话超时自动失效。会话超时默认 30 分钟（可配置）。
- **业务规则**：
  - 同一账号默认允许多端登录（可配置为单点）。
  - 连续 5 次密码错误则锁定账号 30 分钟（可配置）。

#### F-UM-03 用户信息维护

- **个人中心**：用户可查看与修改：手机号（需短信验证）、邮箱（需邮箱验证）、头像、昵称。
- **管理后台**：管理员可修改：部门、角色、状态、备注；不可直接修改密码（只能重置）。
- **业务规则**：工号、姓名变更需走审批流程（一期可走线下审批后由管理员手动调整）。

#### F-UM-04 密码管理

- **密码强度策略**（默认，可配置）：
  - 长度 ≥ 8 位
  - 必须包含大小写字母、数字
  - 不能与最近 3 次密码相同
  - 90 天到期后强制修改
- **修改密码**：用户输入原密码 + 新密码 + 确认新密码。
- **找回密码**：通过注册手机号或邮箱接收验证码重置。
- **管理员重置**：管理员可重置任意用户密码为系统生成的临时密码，下次登录强制修改。

#### F-UM-05 角色管理

- **功能要点**：
  1. 新增/修改/删除角色（默认角色不可删除）。
  2. 为角色分配权限（菜单 + 操作 + 数据范围）。
  3. 查询角色下的用户清单。
- **业务规则**：
  - 删除角色前必须将该角色下的用户先转移或解绑。
  - 角色编码全局唯一，不可修改。

#### F-UM-06 权限管理

采用 **RBAC（基于角色的访问控制）** 模型：

```
用户 ←→ 角色 ←→ 权限
                  ├── 菜单权限（可见的菜单）
                  ├── 操作权限（按钮/接口级）
                  └── 数据权限（仅本部门 / 本人 / 全部）
```

- **菜单权限**：控制左侧导航菜单的可见性。
- **操作权限**：控制具体按钮（如"删除"、"审核"）和后端 API 接口的访问。
- **数据权限**：控制可见的数据范围（如仓库管理员只能看自己负责的仓库）。
- **业务规则**：
  - 一个用户可同时拥有多个角色，权限取并集。
  - 数据权限遵循"最小权限原则"，默认仅本人可见。

#### F-UM-07 用户状态管理

| 状态 | 描述 | 可登录 | 备注 |
|------|------|--------|------|
| 待激活 | 账号已建尚未激活 | 否 | 创建后默认状态 |
| 启用 | 正常使用 | 是 | |
| 禁用 | 管理员手动停用 | 否 | 用于临时停用（如休假、调岗中） |
| 锁定 | 多次密码错误自动锁定 | 否 | 锁定时间结束后自动解除，或管理员手动解锁 |
| 注销 | 离职后注销 | 否 | 数据保留但不可登录，不可恢复 |

#### F-UM-08 部门/组织架构管理

- 支持多级部门树（建议 ≤ 5 级），可新增、编辑、移动、停用部门。
- 部门下挂用户，部门停用前需将用户先迁出。

#### F-UM-09 用户查询与批量操作

- **查询条件**：账号、姓名、手机号、邮箱、部门、角色、状态、创建时间区间。
- **批量操作**：批量启用/禁用、批量分配角色、批量分配部门、批量重置密码、批量导出（CSV/Excel）。
- **导入**：支持 Excel 模板批量导入用户，导入前校验，失败行可下载错误清单。

#### F-UM-10 操作审计日志

- **记录范围**：登录/登出、密码修改、角色/权限变更、用户状态变更、批量操作、敏感数据导出。
- **日志字段**：操作人、操作时间、来源 IP、操作类型、操作对象、操作前后值、操作结果。
- **保留期**：默认 ≥ 12 个月，可配置；过期后归档而非删除。
- **查询**：按操作人、时间、操作类型组合查询；支持导出。

#### F-UM-11 登录安全策略

- **图形验证码**：登录失败 ≥ 2 次后强制图形验证码。
- **登录限频**：同 IP 1 分钟内 ≥ 10 次登录尝试触发限频（拒绝 5 分钟）。
- **双因素认证（2FA）**：可对管理员角色强制开启短信/TOTP 二次验证（一期可选实现）。
- **异地登录提醒**：检测到非常用地登录时邮件/短信通知本人。

### 4.4 用户管理核心业务流程

**4.4.1 新员工开户流程**

```
HR 提交入职信息
   ↓
系统管理员创建账号（指定部门、初始角色）
   ↓
系统下发初始密码（邮件/短信）
   ↓
员工首次登录 → 强制修改密码
   ↓
账号正式启用
```

**4.4.2 离职账号注销流程**

```
HR 提交离职申请
   ↓
系统管理员将账号置为"禁用"（保留 30 天观察期）
   ↓
30 天后无异议，将账号置为"注销"
   ↓
解除所有角色，记录归档
```

**4.4.3 权限申请变更流程**

```
用户提出申请（线上工单）
   ↓
直属上级审核
   ↓
系统管理员执行变更
   ↓
变更日志留存，通知申请人
```

### 4.5 用户管理业务规则汇总

| 规则编号 | 规则描述 |
|----------|----------|
| BR-UM-01 | 工号、手机号、邮箱在系统内唯一 |

| BR-UM-03 | 超级管理员账号不可被禁用、不可被注销，至少保留 1 个 |

| BR-UM-05 | 操作审计日志不可篡改、不可删除 |

## 5. 库存管理模块需求

### 5.1 模块目标

实现商品全生命周期的库存管理，覆盖基础档案、出入库业务、调拨、盘点、预警、查询与报表，确保库存账实一致、流转可追溯、决策有数据支撑。

### 5.2 功能清单

| 编号 | 功能名称 | 优先级 |
|------|----------|--------|
| F-IM-01 | 商品（SKU）档案管理 | P0 |
| F-IM-02 | 仓库与库位管理 | P0 |
| F-IM-03 | 入库管理 | P0 |
| F-IM-04 | 出库管理 | P0 |
| F-IM-05 | 库存调拨 | P1 |
| F-IM-06 | 库存盘点 | P0 |
| F-IM-07 | 库存预警（安全/上限/滞销） | P0 |
| F-IM-08 | 库存查询 | P0 |
| F-IM-09 | 库存报表 | P1 |
| F-IM-10 | 库存流水（台账） | P0 |
| F-IM-11 | 商品分类与单位管理 | P0 |

### 5.3 详细功能需求

#### F-IM-01 商品（SKU）档案管理

- **字段**：商品编码（唯一）、商品名称、分类、规格、品牌、计量单位、条形码、安全库存、库存上限、保质期（天）、是否启用、备注、商品图片。
- **操作**：新增、编辑、停用（不允许删除）、批量导入、批量导出、查询。
- **业务规则**：
  - 商品编码唯一，由系统按规则自动生成或人工录入（可配置）。
  - 已存在库存或历史业务的商品，仅可"停用"，不可"删除"。
  - 关键字段（编码、单位）一旦有业务数据，不允许再修改。

#### F-IM-02 仓库与库位管理

- **仓库**：编码、名称、类型（成品仓/原料仓/退货仓/不良品仓）、地址、负责人、状态。
- **库位**：所属仓库、库位编码、库位名称、容量上限、状态。
- **业务规则**：
  - 一个商品可在多个仓库/库位存放，库存按"仓库 + 库位 + 商品"维度记账。
  - 仓库停用前必须库存清零并无在途单据。

#### F-IM-03 入库管理

- **入库类型**：采购入库、退货入库、调拨入库、盘盈入库、其他入库。
- **入库单字段**：入库单号（自动生成）、入库类型、来源单据号（如采购单号）、目标仓库/库位、商品明细（SKU + 数量 + 单价）、经办人、入库时间、备注、附件。
- **状态流转**：草稿 → 待审核 → 已审核（实际过账，库存增加）→ 已完成 / 已作废
- **业务规则**：
  - 入库单审核后才会增加库存数量。
  - 已审核的单据不可修改，只能作废（生成红冲单）。
  - 入库时校验库位容量上限，超出则提示。
  - 自动生成库存流水。

#### F-IM-04 出库管理

- **出库类型**：销售出库、领用出库、调拨出库、盘亏出库、报损出库、其他出库。
- **出库单字段**：出库单号、出库类型、目标单据号（如销售单号）、源仓库/库位、商品明细、申请人、经办人、出库时间、备注。
- **状态流转**：草稿 → 待审核 → 待出库 → 已出库（库存减少）→ 已完成 / 已作废
- **业务规则**：
  - 出库前校验"可用库存"（在库 - 已锁定 - 在途出库），不足则禁止提交。
  - 支持按 FIFO（先进先出）/ FEFO（先到期先出）/ 指定批次出库（可配置）。
  - 已出库单据不可修改，需作废重做。

#### F-IM-05 库存调拨

- **调拨单**：调出仓库/库位、调入仓库/库位、商品明细、调拨原因、申请人。
- **状态流转**：草稿 → 待审核 → 待调出（调出仓库扫码确认）→ 在途 → 已调入（调入仓库确认收货）→ 已完成
- **业务规则**：
  - 调出后库存进入"在途"状态，不计入调出仓可用库存，也不计入调入仓可用库存。
  - 调入确认时支持差异处理（短少需走"调拨差异"单）。

#### F-IM-06 库存盘点

- **盘点类型**：全盘、抽盘、动态盘点。
- **盘点单字段**：盘点单号、盘点范围（仓库/库位/分类）、盘点人、盘点时间、明细（SKU + 账面数 + 实盘数 + 差异）。
- **流程**：
  1. 创建盘点单（系统冻结相关库存的出入库操作或仅冻结待盘 SKU）。
  2. 仓管员实地盘点录入实盘数。
  3. 系统计算差异，生成"盘盈入库单"或"盘亏出库单"。
  4. 主管审核后过账，库存调整完成。
- **业务规则**：
  - 盘点期间默认禁止盘点范围内 SKU 的出入库（可配置允许）。
  - 差异需填写原因，留档审计。

#### F-IM-07 库存预警

- **预警类型**：
  1. **低库存预警**：实际可用库存 ≤ 安全库存 → 提醒补货。
  2. **超库存预警**：实际库存 ≥ 库存上限 → 提醒控制采购。
  3. **滞销预警**：N 天（默认 90 天，可配置）无出库 → 提醒清理。
  4. **临期预警**：距保质期 ≤ M 天（默认 30 天）→ 提醒优先出库。
- **预警渠道**：站内消息、邮件、短信（可配置）。
- **业务规则**：每日定时（如凌晨 02:00）跑批；可手动触发。

#### F-IM-08 库存查询

- **维度**：按商品、按仓库、按库位、按分类、按批次组合查询。
- **关键指标**：在库数量、可用数量、锁定数量、在途入库、在途出库、安全库存、上限。
- **导出**：支持 Excel 导出。

#### F-IM-09 库存报表

| 报表 | 说明 |
|------|------|
| 库存日报/月报 | 期初 + 入库 - 出库 = 期末，按仓库/分类汇总 |
| 出入库明细报表 | 按时间区间、类型、商品筛选 |
| 库存周转率报表 | 按 SKU/分类计算周转率与周转天数 |
| 滞销库存报表 | N 天无出库的 SKU 列表 |
| 临期库存报表 | 即将到期的 SKU 列表 |
| 盘点差异报表 | 盘点差异统计与原因汇总 |

#### F-IM-10 库存流水（台账）

- 所有改变库存数量的操作（入库、出库、调拨、盘点、调整）必须生成一条流水。
- **字段**：流水号、商品、仓库/库位、变更类型、变更前数量、变更数量、变更后数量、单据号、操作人、操作时间。
- **特性**：流水不可修改、不可删除，仅可追加。

#### F-IM-11 商品分类与单位管理

- **分类**：支持多级分类树（建议 ≤ 4 级）。
- **单位**：维护基础计量单位（个、箱、千克等），支持单位换算（如 1 箱 = 24 个）。

### 5.4 库存管理核心业务流程

**5.4.1 采购入库流程**

```
采购员/系统接收采购到货信息
   ↓
仓管员创建入库单（关联采购单）
   ↓
实物清点入库（按库位上架）
   ↓
仓库主管审核入库单
   ↓
系统增加库存数量、生成流水
   ↓
通知采购、财务
```

**5.4.2 销售出库流程**

```
销售订单生成（外部系统/手工录入）
   ↓
仓管员创建出库单（系统校验可用库存并锁定）
   ↓
拣货、复核、打包
   ↓
仓库主管审核 → 实际出库
   ↓
系统减少库存、生成流水、解除锁定
   ↓
通知销售、客户
```

**5.4.3 库存盘点流程**

```
仓库主管创建盘点单（指定范围）
   ↓
系统冻结相关 SKU 的出入库
   ↓
仓管员实地盘点并录入实盘数
   ↓
系统自动计算差异
   ↓
仓库主管审核差异原因
   ↓
生成盘盈入库单/盘亏出库单
   ↓
过账，解除冻结
```

**5.4.4 库存调拨流程**

```
申请人创建调拨单
   ↓
仓库主管审核
   ↓
调出仓库出库 → 库存进入"在途"
   ↓
调入仓库收货确认（差异处理）
   ↓
库存调整到位，单据完成
```

### 5.5 库存管理业务规则汇总

| 规则编号 | 规则描述 |
|----------|----------|
| BR-IM-01 | 库存数量按"仓库 + 库位 + 商品 + 批次（可选）"维度记账 |
| BR-IM-02 | 任何改变库存数量的操作必须生成流水，流水不可修改 |
| BR-IM-03 | 出库前必须校验可用库存，不足禁止出库 |
| BR-IM-04 | 已审核的业务单据不可修改，只能作废重做 |
| BR-IM-05 | 库存数据按"实时账面 + 异步流水"双写，确保一致 |
| BR-IM-06 | 商品有库存或业务时不可删除，只可停用 |
| BR-IM-07 | 调拨在途库存不计入任何仓库的可用库存 |
| BR-IM-08 | 盘点期间默认锁定范围内 SKU 的库存变更 |
| BR-IM-09 | 安全库存、库存上限按"仓库 + 商品"维度配置 |
| BR-IM-10 | 所有库存数量必须 ≥ 0，系统层面禁止负库存 |

## 6. 非功能性需求

### 6.1 性能需求

| 指标 | 要求 |
|------|------|
| 普通页面响应时间 | ≤ 1.5 秒（95 分位） |
| 关键查询接口响应 | ≤ 500ms（95 分位） |
| 并发用户数 | ≥ 500 在线，≥ 50 并发操作 |
| 库存数据写入延迟 | ≤ 1 秒 |
| 报表生成 | 万行数据 ≤ 10 秒 |
| 批量导入 | 1 万行 ≤ 60 秒 |

### 6.2 安全需求

- 密码采用 bcrypt/argon2 加盐哈希存储，禁止明文。
- 传输全程 HTTPS。
- 敏感字段（手机号、邮箱）在前端展示时支持脱敏。
- 防 SQL 注入、XSS、CSRF。
- 权限校验前后端双重，后端为准。
- 关键操作（删除、批量、权限变更）需二次确认 + 审计。
- 登录会话采用 Token + 刷新机制，支持主动撤销。

### 6.3 可用性需求

- 系统年可用性 ≥ 99.5%（计划维护除外）。
- 关键数据每日备份，保留 ≥ 30 天。
- 故障 RTO ≤ 4 小时，RPO ≤ 1 小时。

### 6.4 可维护性与可扩展性

- 模块化设计，用户管理、库存管理可独立部署演进。
- 提供配置中心，业务规则（如密码策略、预警阈值、超时时间）可配置化。
- 数据字典与枚举可后台维护。
- 预留对外 API（OpenAPI 规范），便于后续与 ERP/CRM/财务系统对接。

### 6.5 兼容性需求

- 浏览器：Chrome / Edge / Safari 最新两个大版本。
- 屏幕分辨率：≥ 1366×768。
- 字符集：UTF-8（支持中文、英文、emoji）。

## 9. 风险与假设

### 9.1 主要风险

| 风险编号 | 描述 | 影响 | 应对策略 |
|----------|------|------|----------|
| R1 | 历史库存数据不准，迁移后账实不符 | 高 | 上线前组织一次全盘，校准初始数据 |
| R2 | 仓库人员对系统不熟悉，作业效率短期下降 | 中 | 提前培训 + 上线初期驻场支持 2 周 |
| R3 | 权限模型设计与实际组织不匹配 | 中 | 上线前与各部门主管共评权限矩阵 |
| R4 | 与 HR 系统数据同步异常导致账号差异 | 中 | 设计同步对账机制 + 人工兜底入口 |
| R5 | 高并发出库时锁库存性能瓶颈 | 中 | 数据库行锁 + 乐观锁双策略，压测验证 |

### 9.2 假设条件

- 公司 HR 系统能提供员工入离职数据接口或定期导出。
- 仓库具备基本网络条件，未来可逐步引入 PDA 扫码。
- 一期不涉及多币种、多语言、跨境业务。
